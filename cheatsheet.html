Introduction

Infamous first words

The Concept

The design concept applied here is not exactly rocket science. Basically, it is a combination of a simplified grid (i.e., an outrageously stripped down “960” approach), and the “Goldilocks Approach to Responsive Web Design” as proposed by Chris Armstrong.

Websites should always cater to users rather than devices. Thus, the “GL Approach” is utterly device agnostic. Its primary goal is to keep lines of text as close as possible (and the screen estate allows) to the width recommended for easy reading (appr. 30em or 60 characters).

So unless vendors decide to come out with round screens, or you have to create fancy layouts, you should be covered.

The idea behind this approach is to give the developer a quick–and–dirty solution, but enough wiggle room to position elements to effect. A very convenient side–effect is the massive reduction of necessary code to accomplish the task.

Visiting quite a number of one–page websites (a.k.a. “landing pages”) of late, I figured that most developers appear to be content with a rather small variety of design options; many of the commands their style sheets provided, were simply not used. This brought me to cut down the lot to the most popular ones: rows that cover the entire width, or are shared by two or three elements. For variety (or special design needs, really), I added two more grid elements (see in “How to Use this Website”).

How to Use this Website

First of all, make a copy of the unzipped repository, and store it in a safe place. Always work off a copy. Don’t mess with the original. Crap happens to the best of us, so it’s always nice to have a clean copy ready.

What code the cheat sheet contains may be selected (and by such, will in some browsers be copied to the clipboard — if not, press the keys CTRL and C (⌘ and C) — by using the respective button placed on top of individual blocks of code.

This should save you from having to run your pointing device across the screen (or hack away until your arrow keys melt, if you are using the keyboard), and make things a tad easier for users of mobile devices. To paste the clipboard’s content into a document, press CTRL and V (⌘ and V).

To create your own grid, use the following CSS classes:

.row
positions the grid elements it contains side–by–side (as screen estate allows)
.grid-4, .grid-6, and .grid-12
determines the width of displayed columns; take 12 and divide it by the grid number to get the number of columns in any one row
.grid-2 and .grid-5
these two rules may be convenient, if you need to go beyond a simple grid to create your design

Little boxes made of ticky–tacky

HTML Structure

This repository contains no index.html, so you may use the unzipped repository as a project root (and launch the cheatsheet.html in a browser tab next to your project files). This should help keep confusion at a minimum.

If you haven’t done so yet, create an empty document and save it as index.html in your project directory (folder), commonly referred to as “root”. Copy the HTML structure below and paste it into the newly created document. Launch this document in a browser, the title given in the source code should be visible in the top bar of your browser tab.

CSS Structure

The CSS directory of this repository already contains two empty files, called 30max.css and 30min.css respectively.

Copy the CSS commands below into the individual files. Unless you decided to change any references in your HTML source code, you are set to begin customising.

Remember that media queries act like filters: Commands inside a particular media query apply to devices meeting the specification set in the query only, while commands outside a media query are passed on to devices responding to subsequent specifications, too. No worries, it’s easier than it sounds.

Typography

A matter of style

Typographical HTML–Tags

This section contains the most common typographical HTML–tags. I chose to display them “in place” rather than separating text from code. Screenreader users: I’m sorry for the inconvenience this may cause, but this is a cheat sheet rather than a regular website.

In general, HTML–tags come in pairs, wrapping content. Exceptions are few (16, for all I know). Those you will frequently come across are even fewer:

  • meta information (<meta … />)
  • images (<img … />)
  • form fields (<input … />)
  • line breaks (<br />)
  • and, if all fails, horizontal rulers (<hr />)

Tags not coming in pairs have to be self–closing (reference: <a title="See example of self–closing tag" href="#media">code “Images”<a> below).

Natively, HTML knows six orders of headlines: <h1> … </h1> to <h6> … </h6>

Most commonly, single–page websites use one <h1>Headline of the First Order</h1> for its title. <h2>Headlines of the Second Order</h2> make excellent headers for your sections. And <h3>Headlines of the Third Order</h3> may be used to head smaller (less important) elements.

If you believe to need more than these in a single–page website, you may want to reconsider your site structure. Note that headlines are tools to structure your content rather than decorate it, so use them in correct hierarchical order (highest to lowest in linear order).

<p>By default, paragraphs do have “leading” (i.e., horizontal margins that separate them visually), but in reset.css all margins have been set to 0 (zero). So they will have to be set again in your custom CSS. Yes, this makes sense!</p>

You may <em>semantically emphasise</em> words or phrases, <strong>visually emphasise</strong> them, or <mark>mark keywords</mark>. It is even possible to combine (but not mix) them, but know that you will make hardboiled typographers hang their heads and weep like babies, if you do. When nesting tags, the rule “first in, last out” applies. Consult the source code, you will instantly recongnise a pattern.

Abbreviations (like, <abbr title="HyperText Markup Language">HTML</abbr>) and acronyms (e.g., <abbr title="Keep It Simple, Stupid">KISS</abbr>) share the same tag. To explain proper (but exotic) words, you may use “spans” instead: <span title="a lover of rain; someone who finds joy and peace of mind during rainy days">Pluviophile</span>.

Typographical Notes

If you are using utf-8 character coding, typographical characters should be displayed just fine in most browsers. I have yet to hear of basic punctuation (exclamation marks, questions marks, commas, colons, semicolons, and periods a.k.a. full stops) causing any problems; other characters may be tricky, though.

You may want to develop the habit of using “HTML entities” or “Decimal code” instead of the respective key (or shortcut) on your keyboard. Basically, there is no reason or rule against mixing HTML entities and decimal code (but in the long run, you will find it more useful to stick with one of the systems). In general, decimal code is considered to be the most stable solution; at any rate it provides the widest range of special characters. For details, consult <a rel="external" href="https://smartquotesforsmartpeople.com/">“Smart Quotes for Smart People”<a>.

All hyphens are dashes, but not all dashes are hyphens. Use an “n–dash” (&ndash;) to divide compound words or syllables. Yet to set off parenthetical statements (parts of sentences), you should use the “m–dash” (&mdash;). This sentence contains — in order of appearance — two “m–dashes” and two “n–dashes”.

Whether or not to set empty spaces before and after “m–dashes” is less a matter of style than one of cultural preference (where you come from, in other words).

To display the trademark (™), the registered (®), or the copyright sign (©), use &trade;, &reg; and &copy;, respectively.

The typographical character you will (probably) come across in source code most frequently is the ampersand (&). That’s because it is the most versatile character: it may be displayed (to substitute the word “and”) and processed (not displayed, but used to display other special characters and, of course, itself).

To display it in a stable way, type &amp; or &#38; (note that both codes contain the character itself). Since it looks awkward to have a single character displayed before or after a line break, you may want to combine it with non–breaking spaces in name compounds (or omit white spaces).

Examples: Black&nbsp;&amp;&nbsp;White will set off the ampersand, but still keep the compound from suffering a line break (Black & White), while Black&amp;White will keep the compound intact, without any white spaces (Black&White).

Note: Using the ampersand as a substitute for the word “and” in running text is considered poor style. By the way, it’s Ben & Jerry’s® (ice cream), but Tom and Jerry (cartoon characters).

Nested Lists

Provided that cleaning is conducted rather regularly, it will make a lot of sense to start with shopping (especially when the fridge is empty), then do the cooking, and eventually the cleaning. That was easy, but rather boring. Let’s do something that reads not only like a plan but like a strategy in the making:

  1. shopping
    • baker
    • butcher
    • greengrocer
  2. cooking
    1. tart
    2. salad
    3. steaks
  3. cleaning
    1. cutlery
    2. plates
    3. pans
    4. floor

This list appears to be a bit elaborate, but we are looking at four lists, really. Nevertheless, the meaning couldn’t be any clearer. Shopping, first priority (order of shops irrelevant, as long as they are located close together). Then cooking (first the tart, because it takes the longest; then the salad; and finally the steaks, because they will be finished in no time). Eventually, we need to clean up the mess. Here, the particular order is important, because we want to save dishwater, and don’t want to splash greasy water on a clean floor. Accidents may happen, after all.

Media

Images, videos, and audio files

The Groovy Lemon Pie

Images, videos, and audio files can greatly improve the functionality of your website. Yet there will be hardly an element as troublesome to your visitors (and also to you), when implemented without reason or care.

Even purely visual elements (eye candies without a function other than being pretty) should come with options. You have no way of seriously anticipating who will visit your website, or how, so see to give your visitors alternatives. Don’t waste their bandwidth just because you can …

/* MEDIA */
/* next line: images and native videos are responsive (to the given screen size) */
img, video { max-width: 100%; height: auto; }
/* next two lines: videos are displayed 16:9 and iframes do not fall back to default; default size (width and height) of youtube’s embedding code was removed */
.youtube { position: relative; padding-bottom: 56.25%; height: 0; overflow: hidden; max-width: 100%; }
.youtube iframe { position: absolute; top: 0; left: 0; width: 100%; height: 100%; }

Outsourcing Services

Stay in touch with your visitors

Contact Form

Using a contact form on a static server is quite easy with Formspree. There may be more services of that kind out there, but Formspree was the first I found, and I’m quite happy with it.

Copy the form they provide into your source code and replace the string after their web address in the “action” attribute (the “your@email.com” part, that is) with your e-mail address. That’s it.

<form class="contactform" action="http://formspree.io/your@email.com" method="POST">
<p><label for="name">Name <input type="text" id="name" name="name" placeholder="Name (required)" required /></label></p>
<p><label for="email">E-Mail <input type="email" id="email" name="_replyto" placeholder="E-Mail (required)" required /></label></p>
<p><label for="message">Message <textarea id="message" name="message" rows="4" placeholder="Message (required)" required></textarea></label></p>
<input type="text" name="_gotcha" style="display:none" />
<button id="submit" type="submit" class="button">Send</button>
</form>
